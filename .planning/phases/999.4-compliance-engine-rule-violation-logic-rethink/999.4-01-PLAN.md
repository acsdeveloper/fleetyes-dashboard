# Plan 01: Types & Adapter Foundation

## Objective
Extend `Activity` and `ComplianceViolation` with trip-context fields, and update the adapter
to tag every produced Activity with its source Order UUID. These changes are pre-requisites
for all downstream plans that read the enriched violation shape.

## must_haves
- `Activity` interface contains `orderId?: string`
- `ComplianceViolation` interface contains `tripId?: string`, `tripStartTime?: string`, `tripEndTime?: string`
- `orderToActivity()` in adapter.ts sets `orderId: order.uuid` on every returned object
- All 71 existing tests still pass (`npx vitest run apps/v4/lib/compliance-engine` exits 0)

```ts
export interface ComplianceViolation {
  /** Machine-readable rule identifier, e.g. "GB_GOODS_DAILY_DRIVE_LIMIT" */
  ruleId:      string
  /** Whether this is a hard violation or an approaching-limit warning */
  severity:    ComplianceSeverity

  /** Driver UUID */
  driverUuid:  string
  /** Human-readable explanation */

  /**
   * The exact calculation that triggered this issue.
   * e.g. "10h 32m driving vs 10h limit"
   */

}
```

Add three optional trip-context fields AFTER `ruleset`:

```ts
export interface ComplianceViolation {
  /** Machine-readable rule identifier, e.g. "GB_GOODS_DAILY_DRIVE_LIMIT" */
  ruleId:      string
  /** Whether this is a hard violation or an approaching-limit warning */
  severity:    ComplianceSeverity
  /** The date (YYYY-MM-DD) this issue occurs on */
  date:        string
  /** Driver UUID */
  driverUuid:  string
  /** Human-readable explanation */
  message:     string
  /**
   * The exact calculation that triggered this issue.
   * e.g. "10h 32m driving vs 10h limit"
   */
  calculation: string
  /** Which ruleset this rule belongs to */
  ruleset:     Ruleset
  /**
   * Optional trip context — set when a single trip can be identified as the cause.
   * Absent for aggregate-window rules (weekly totals, fortnightly totals) that span
   * multiple trips. Always set by the prospective check (the new trip IS the cause).
   */
  tripId?:        string   // UUID of the triggering Order/trip
  tripStartTime?: string   // ISO datetime of trip start
  tripEndTime?:   string   // ISO datetime of trip end (or best estimate)
}
```
</action>
