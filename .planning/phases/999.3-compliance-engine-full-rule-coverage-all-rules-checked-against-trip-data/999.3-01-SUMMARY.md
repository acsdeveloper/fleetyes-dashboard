# Summary: Engine Bug Fixes & Type Additions

## What Was Built

### D-10 Fix: `assimilated.ts` daily work limit for NON_DRIVING_DUTY
The critical `if (!day.hasDriving) continue` guard was removed. Non-driving working days now get EU_DAILY_WORK_LIMIT checked:
- ≤ 13h → no issue
- 13h–15h → warning
- > 15h → violation

Two new constants added: `MAX_DUTY_WARN_MINS = 780` and `MAX_DUTY_REDUCED_MINS = 900`.

### D-04 Fix: `prospective-check.ts` MISSING_END_TIME block removed
The 12-line block that pushed a `MISSING_END_TIME` hard violation has been deleted. Trips without end times no longer block assignment. The 2h fallback in `tripEnd()` continues to handle gap calculation gracefully.

### D-09: `types.ts` `breaksAssumedCompliant` flag
Added `breaksAssumedCompliant?: boolean` to `VehicleConfig` with default `true`. This is a forward-compatibility hook for future break enforcement from actual driving timestamps.
