## Task 2 — Final grep audit for remaining hardcoded strings

<read_first>
- All `apps/v4/app/(dashboard)/*/page.tsx` files (search via grep, not by reading individually)
</read_first>

<action>
Run targeted greps to catch remaining hardcoded patterns:

```bash
# Pattern 1: JSX text nodes with English words (most reliable signal)
grep -r --include="page.tsx" -n ">Cancel<\|>Save<\|>Delete<\|>Filter<\|>Search<\|>Upload<\|>Close<\|>Submit<" apps/v4/app/\(dashboard\)/

# Pattern 2: Common hardcoded button labels
grep -r --include="page.tsx" -n '"Cancel"\|"Save"\|"Delete"\|"Filter"\|"Upload"\|"Close"\|"Submit"\|"Apply"\|"Clear all"' apps/v4/app/\(dashboard\)/

# Pattern 3: Placeholder text
grep -r --include="page.tsx" -n 'placeholder="[A-Z]' apps/v4/app/\(dashboard\)/

# Pattern 4: Empty state messages
grep -r --include="page.tsx" -n '"No \|"Nothing \|"No records\|"No data"' apps/v4/app/\(dashboard\)/

# Pattern 5: aria-label with English
grep -r --include="page.tsx" -n 'aria-label="[A-Z]' apps/v4/app/\(dashboard\)/
```

For each hit returned:
1. Determine if it's a user-visible string (not a CSS class, API value, or variable name)
2. If user-visible: find or create the appropriate `t.xxx` key and replace
3. If already covered by a dynamic `t.xxx` in scope: verify it is actually being used (not shadowed)

Any fixes: commit `fix(i18n-999.9): fix remaining hardcoded strings found in final audit`
</action>

<acceptance_criteria>
- Grep for `">Cancel<"` in dashboard pages returns 0 results
- Grep for `">Save<"` in dashboard pages returns 0 results
- Grep for `'placeholder="[A-Z]'` in dashboard pages returns 0 results (or only non-user-facing technical placeholders)
- Grep for `'"No records found"'` in dashboard pages returns 0 results
</acceptance_criteria>
